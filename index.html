Ventajas de NoSQL:

  • Object-oriented programming.
  • Flexible.
  • Volúmen de datos.
  • Velocidad de proceso.
  • Escalabilidad.
  • Schemas mutables.

Desventajas de NoSQL:

  • Normalización de estándars.
  • Atomicidad de las instrucciones.
  • Flexibilidad.
  • Schemas mutables.
  • Las tareas/consultas las debe hacer tu aplicación.
  • Renunciar a SQL.
  • No hay ACID.

¿Por qué no necesito una base de datos No relacional?

Cuando usar MongoDB(No SQL)

  • Tu data no tiene sentido.
  • Tu data es tipo: Documentos, Key-value, grafos...
  • Para obtener un dato, usas múltiples JOINs...
  • Estás rompiendo con el esquema relacional.
  • La persistencia en los datos, no es prioridad.
  • La estructura de datos que manejamos son variables.
  • La necesidad del proceso, no se prevee.
